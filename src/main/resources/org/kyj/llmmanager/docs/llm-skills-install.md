# LLM 스킬 설치

LLM Manager에 포함된 기본 스킬과 룰을 프로젝트에 설치하는 화면이다.

---

## 화면 구성

| 영역 | 설명 |
|------|------|
| 프로젝트 경로 | 설치 대상 프로젝트 루트를 지정한다. |
| 프로젝트명 | 프로젝트 히스토리와 변수 치환에 사용된다. |
| 언어 | 템플릿 변수 `language` 값으로 사용된다. |
| 작성자 | 템플릿 변수 `author` 값으로 사용된다. |
| 설치할 LLM 도구 | 설치 가능한 도구 목록이다. |
| 스킬 팩 선택 | 선택한 도구의 팩 목록이다. |
| 미리보기 | 선택된 파일들의 설치 예정 내용을 보여준다. |

---

## 기본 흐름

1. `프로젝트 경로`를 선택한다.
2. 필요한 LLM 도구를 선택한다.
3. 각 도구에서 설치할 스킬 팩을 선택한다.
4. `설치` 또는 `백업 후 설치`를 누른다.

---

## 동작

- `설치`를 누르면 선택된 파일이 프로젝트 경로에 복사된다.
- 동일한 파일이 이미 있으면 덮어쓴다.
- `백업 후 설치`를 누르면 기존 파일을 `.llm-backup/` 아래에 먼저 저장한 뒤 설치한다.
- 설치가 끝나면 `최근 프로젝트` 목록에 현재 프로젝트가 기록된다.
- 같은 프로젝트 경로로 다시 설치하면 `projects.json` 기준으로 기존 항목을 갱신한다.

---

## 변수 치환

설치 파일이 템플릿인 경우 아래 변수를 치환한다.

| 변수 | 의미 |
|------|------|
| `projectName` | 프로젝트명 |
| `language` | 선택한 언어 |
| `author` | 작성자 |

---

## 히스토리

- `최근 프로젝트` 목록은 이전에 설치한 프로젝트 경로를 보여준다.
- 항목을 선택하면 해당 프로젝트 값이 폼에 복원된다.
- 프로젝트 경로가 같으면 중복으로 쌓이지 않고 최신 항목으로 갱신된다.

---

## 자주 쓰는 점검

- 프로젝트 경로가 비어 있으면 설치할 수 없다.
- 도구와 팩을 하나도 선택하지 않으면 설치할 파일이 없다.
- 설치 전 미리보기에서 선택 상태를 확인하는 것이 좋다.
