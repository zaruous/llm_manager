# LLM Wiki Agent

문서를 수집하면 AI가 분석해 상호 연결된 지식 위키를 자동으로 구축·유지하는
에이전트입니다. RAG와 달리 위키가 지속적으로 누적되며, 모순 검출과 지식 그래프
시각화를 지원합니다.

---

## 시작하기

### 1. 워크스페이스 만들기

위키 한 개 = 디렉토리 한 개입니다. 두 가지 방법 중 하나로 만듭니다.

- **문서 수집 다이얼로그에서 자동 생성** — 빈 디렉토리를 워크스페이스로 지정하고
  수집을 시작하면 골격 초기화를 제안합니다 (가장 간단)
- **LLM 스킬 설치** 화면에서 `LLM Wiki Agent > 위키 워크스페이스 골격` 팩 설치

```
my-wiki/
├── raw/      ← 원본 문서 (불변)
├── wiki/     ← 위키 페이지 (에이전트가 관리)
│   ├── index.md
│   └── log.md
└── graph/    ← 지식 그래프 산출물
```

### 2. 사용 방식 선택

| 방식 | 비용 | 준비물 |
|------|------|--------|
| **A. Claude Code 스킬** (권장) | 구독에 포함 | `위키 Claude Code 스킬` 팩 설치 후 해당 폴더에서 `claude` 실행 |
| **B. 플러그인 메뉴** | API 종량제 과금 | Python 3.8+, `pip install litellm`, `ANTHROPIC_API_KEY` 환경변수 |

A 방식은 스킬 팩 설치 후 Claude Code에서 `/wiki-ingest`, `/wiki-query` 등을
바로 사용합니다. B 방식은 아래 플러그인 메뉴를 사용합니다.

---

## 플러그인 메뉴 (B 방식)

`플러그인 > LLM Wiki Agent` 메뉴에서 실행합니다.

| 메뉴 | 동작 | LLM 호출 |
|------|------|---------|
| 문서 수집 (Ingest) | 파일·폴더를 멀티 선택해 `raw/분류/`로 복사 후 위키 페이지 생성 | 발생 (파일 수 비례) |
| 위키 질의 (Query) | CLI 세션처럼 질문·응답. 이력은 워크스페이스에 저장되어 복원됨 | 발생 |
| 구조 검사 (Health) | 빈 페이지·인덱스 불일치·수집 로그 누락 검사 → `wiki/health-report.md` | 없음 |
| 품질 검사 (Lint) | 모순·낡은 내용 검사 → `wiki/lint-report.md` | 발생 |
| 그래프 빌드 (Graph) | 지식 그래프 생성 → `graph/graph.html` | 발생 |
| 그래프 열기 | 빌드된 그래프를 OS 기본 브라우저로 열기 | 없음 |
| 위키 브라우저 | 트리 + 마크다운 뷰로 페이지 탐색, [[위키링크]] 이동, 키워드 검색(제목·본문) | 없음 |

> 질의는 매번 독립 실행됩니다. 이전 문답은 화면 이력일 뿐 다음 질의의
> 컨텍스트로 전달되지 않습니다. 연속성이 필요하면 응답을 `--save`로 위키에
> 저장하세요 — 다음 질의가 위키에서 읽어갑니다.

## 설정

`설정 > Wiki Agent` 탭:

- **실행 에이전트** (드롭다운) — `claude`(기본) 또는 `cursor`
  - `claude`: Claude API(litellm) 기반 도구 직접 실행. Python 3.8+ / `pip install litellm` /
    `ANTHROPIC_API_KEY` 필요
  - `cursor`: 작업을 **Cursor Agent에 위임**. Python 불필요, `CURSOR_API_KEY`와
    cursor-agent-runner 플러그인(Node.js 22+, @cursor/sdk) 필요.
    워크스페이스에 AGENTS.md(위키 워크플로우 스키마)가 없으면 자동 설치됨
- **ANTHROPIC_API_KEY** — claude 에이전트용. 시스템 환경변수로 설정 (secret은 저장되지 않음)
- **LLM_MODEL / LLM_MODEL_FAST** — claude 에이전트용 모델. 기본값 `claude-sonnet-4-6` / `claude-haiku-4-5`
  (cursor 에이전트는 Cursor 설정 탭의 Default Model을 따름)
- **기본 워크스페이스** — 다이얼로그가 처음 열릴 때 사용할 디렉토리.
  수집·질의·브라우저에서 워크스페이스를 선택하면 **자동으로 기억**되어 다음에 복원됩니다
- **실행 타임아웃 (분)** — 비우면 **무제한**(기본). 숫자를 넣으면 그 시간 초과 시
  프로세스를 자식까지 포함해 강제 종료합니다

## 실행 제어

- 수집·질의 다이얼로그의 **중지** 버튼으로 실행 중인 작업을 즉시 종료할 수 있습니다
  (Cursor 위임 실행 포함, 자식 프로세스까지 정리됨)

## 자주 묻는 문제

| 증상 | 원인·해결 |
|------|----------|
| `No module named litellm` | `pip install litellm` 실행 |
| `위키 워크스페이스가 아닙니다` | wiki/index.md 필요 — 문서 수집 다이얼로그가 초기화를 제안하며, 수락하면 자동 생성됨 (스킬 설치 화면 불필요) |
| PDF·DOCX 수집 실패 | `pip install markitdown` (비마크다운 변환기) |
| 그래프 커뮤니티 색상 없음 | `pip install networkx` (선택) |
| 워크스페이스 `tools/`가 덮어써짐 | 정상 동작 — 플러그인이 매 실행마다 최신 도구를 동기화합니다 |
