# LLM Wiki Agent

문서를 수집하면 AI가 분석해 상호 연결된 지식 위키를 자동으로 구축·유지하는
에이전트입니다. RAG와 달리 위키가 지속적으로 누적되며, 모순 검출과 지식 그래프
시각화를 지원합니다.

---

## 시작하기

### 준비물

플러그인 메뉴에서 위키 작업을 실행하려면 아래 항목이 필요합니다.

- `CURSOR_API_KEY` 시스템 환경변수
- `cursor-agent-runner` 플러그인
- Node.js 22 이상과 글로벌 `@cursor/sdk`
- 대화 수집 시 `Wiki MCP Server` (`enable-write=true`)

Claude Code에서 직접 사용할 때는 `LLM Wiki Agent > 위키 Claude Code 스킬` 팩을
설치한 뒤 위키 워크스페이스에서 `claude`를 실행합니다.

### 1. 워크스페이스 만들기

위키 한 개 = 디렉토리 한 개입니다. 두 가지 방법 중 하나로 만듭니다.

- **문서 수집 다이얼로그에서 자동 생성** — 빈 디렉토리를 워크스페이스로 지정하고
  수집을 시작하면 골격 초기화를 제안합니다 (가장 간단)
- **LLM 스킬 설치** 화면에서 `LLM Wiki Agent > 위키 워크스페이스 골격` 팩 설치

```
my-wiki/
├── raw/      ← 원본 문서 (불변)
├── wiki/     ← 위키 페이지 (에이전트가 관리)
│   ├── index.md
│   └── log.md
└── graph/    ← 지식 그래프 산출물
```

### 2. 사용 방식 선택

| 방식 | 비용 | 준비물 |
|------|------|--------|
| **A. Claude Code 스킬** (권장) | 구독에 포함 | `위키 Claude Code 스킬` 팩 설치 후 해당 폴더에서 `claude` 실행 |
| **B. 플러그인 메뉴** | Cursor API 과금 | `CURSOR_API_KEY`, cursor-agent-runner 플러그인, Node.js 22+, @cursor/sdk |

A 방식은 스킬 팩 설치 후 Claude Code에서 `/wiki-ingest`, `/wiki-query` 등을
바로 사용합니다. B 방식은 아래 플러그인 메뉴를 사용하며, 모든 작업이
**Cursor Agent에 위임**되어 실행됩니다.

---

## 플러그인 메뉴 (B 방식)

`플러그인 > LLM Wiki Agent` 메뉴에서 실행합니다. 그래프 열기·위키 브라우저를
제외한 모든 메뉴는 Cursor Agent가 워크스페이스의 AGENTS.md 워크플로우에 따라
수행합니다 (AGENTS.md가 없으면 자동 설치됨).

| 메뉴 | 동작 | Cursor Agent 호출 |
|------|------|---------|
| 문서 수집 (Ingest) | 파일·폴더를 멀티 선택해 `raw/분류/`로 복사(기존 파일은 덮어쓰지 않음) 후 작업 계획에 따라 나눠 수집 — 작은 파일은 5개씩 배치, 큰 텍스트(200KB↑)는 섹션 노트→병합 시퀀스, 큰 PDF(4MB↑)는 페이지 이미지+텍스트로 전처리해 5페이지씩 이미지 판독→병합(`pip install pymupdf` 필요, 없으면 단계 처리 폴백). 진행 상태는 프로그레스바로 표시, 시작 전 실행 횟수 확인. 이미 수집된 파일은 자동 스킵 | 발생 (작업 수 비례) |
| 위키 질의 (Query) | CLI 세션처럼 질문·응답. 이력은 워크스페이스에 저장되어 복원됨 | 발생 |
| 구조 검사 (Health) | 빈 페이지·인덱스 불일치·수집 로그 누락 검사 → `wiki/health-report.md` | 발생 |
| 품질 검사 (Lint) | 모순·낡은 내용 검사 → `wiki/lint-report.md` | 발생 |
| 그래프 빌드 (Graph) | 지식 그래프 생성 → `graph/graph.html` (`tools/build_graph.py` 활용 가능) | 발생 |
| 그래프 열기 | 빌드된 그래프를 OS 기본 브라우저로 열기 | 없음 |
| 위키 브라우저 | 트리 + 마크다운 뷰로 페이지 탐색, [[위키링크]]·상대 `.md` 링크 이동, 키워드 검색(제목·본문) | 없음 |

> 질의는 매번 독립 실행됩니다. 이전 문답은 화면 이력일 뿐 다음 질의의
> 컨텍스트로 전달되지 않습니다. 연속성이 필요하면 질의 창의 저장 옵션으로 응답을
> `wiki/syntheses/` 아래 새 페이지로 저장하세요. 다음 질의는 저장된 위키 내용을 읽어갑니다.

## 설정

`설정 > Wiki Agent` 탭:

- **CURSOR_API_KEY** — 시스템 환경변수로 설정 (secret은 저장되지 않음)
- **기본 워크스페이스** — 다이얼로그가 처음 열릴 때 사용할 디렉토리.
  수집·질의·브라우저에서 워크스페이스를 선택하면 **자동으로 기억**되어 다음에 복원됩니다
- **실행 타임아웃 (분)** — 비우면 **무제한**(기본). 숫자를 넣으면 그 시간 초과 시
  프로세스를 자식까지 포함해 강제 종료합니다

모델·응답 언어는 `설정 > Cursor` 탭의 Default Model / 응답 언어 설정을 따릅니다.
한국어가 기본이며, Cursor Agent의 사고 과정과 응답에도 같은 언어 지시가 적용됩니다.

### 임베딩 및 색인 상태

- 목표 청크 문자 수: 기본 1,500자
- 최대 청크 문자 수: 기본 2,500자
- 청크 중첩 문자 수: 기본 200자
- `선택 디렉토리 색인 상태 확인`: 파일별 최신·변경됨·미색인·빈 문서·잔여 색인 표시

상태 조회는 임베딩 서버를 호출하지 않고 현재 전처리 청크 해시와 SQLite 해시를
비교합니다. `변경됨` 또는 `미색인`이면 플러그인 메뉴의 벡터 색인 갱신을 실행합니다.

## LLM 대화 수집

현재 대화를 수집하려면 `LLM 스킬 설치` 화면에서 사용하는 클라이언트의
`wiki-ingest-conversation` 스킬을 설치하고 Wiki MCP Server를 연결합니다.

1. Wiki MCP Server의 `workspace`를 Wiki 루트로 지정
2. `enable-write=true` 설정
3. Wiki MCP Server 시작
4. MCP 클라이언트에 `http://localhost:18090/mcp` 등록
5. 대화에서 명시적으로 수집 요청

Codex 사용 예:

```text
$wiki-ingest-conversation 현재 대화를 Wiki에 수집해
```

스킬은 관련 user/assistant 대화만 선별하고 시스템 지시, 내부 추론, 불필요한
도구 출력과 비밀값을 제외합니다. 대화 원본은 `raw/mcp-ingest/<날짜>/`에 보존되고
Cursor Agent가 `wiki/` 페이지를 생성·갱신합니다.

자세한 설치 경로, MCP 연결, 상태 확인 방법은 도움말의 `Wiki 대화 수집` 주제를
참조하세요.

## 실행 제어

- 수집·질의 다이얼로그의 **중지** 버튼으로 실행 중인 작업을 즉시 종료할 수 있습니다
  (자식 프로세스까지 정리됨)

## 자주 묻는 문제

| 증상 | 원인·해결 |
|------|----------|
| `필수 환경변수가 없습니다: CURSOR_API_KEY` | 시스템 환경변수에 키 설정 후 앱 재시작 |
| `Cursor Agent Runner 플러그인이 없습니다` | cursor-agent-runner 플러그인 설치 (Node.js 22+, 글로벌 @cursor/sdk) |
| `위키 워크스페이스가 아닙니다` | `wiki/index.md`를 가진 루트 폴더를 선택해야 함. 빈 폴더면 문서 수집 다이얼로그가 초기화를 제안하며, 수락하면 자동 생성됨 |
| 글로벌 @cursor/sdk를 못 찾음 | Node.js와 글로벌 `@cursor/sdk` 설치 상태를 확인한 뒤 앱 재시작 |
| 사이드카 타임아웃 | 기본은 무제한. 설정의 `실행 타임아웃 (분)`이 설정돼 있다면 비우거나 늘리기 |
| 폴더 수집 시 일부 파일이 제외됨 | 정상 동작 — 지원 확장자(md/pdf/docx/html/txt 등)만 재귀 수집하며, 숨김 파일·`.git` 등 도트 디렉토리는 자동 제외됩니다 |
| 큰 파일 수집이 여러 번 실행됨 | 정상 동작 — 200KB 초과 텍스트는 섹션별 노트 작성 후 병합하고, 4MB 초과 PDF는 페이지 이미지를 5페이지씩 판독 후 병합합니다. 임시 파일은 `.llm-manager/ingest/`에 두고 완료 시 자동 정리됩니다 |
| 큰 PDF가 페이지 분할되지 않음 | `pip install pymupdf` 실행 (페이지 전처리 도구 요구사항). 미설치 시 에이전트 단계 처리로 폴백되며 수집 자체는 동작합니다 |
| 동일 이름 파일을 다시 수집함 | raw는 불변 저장소 — 내용이 같으면 기존 파일을 재사용하고, 다르면 `이름-1.확장자`로 새로 저장합니다 |
| 워크스페이스 `tools/`가 덮어써짐 | 정상 동작 — 플러그인이 매 실행마다 최신 도구를 동기화합니다 (그래프 빌드 시 에이전트가 활용) |
| 대화 수집 스킬이 보이지 않음 | 스킬 팩 설치 경로를 확인하고 Codex/Cursor/Claude 세션을 다시 시작합니다 |
| `wiki_ingest` 도구가 없음 | Wiki MCP Server에서 `enable-write=true`로 저장한 뒤 서비스를 재시작합니다 |
| 대화는 수집됐지만 시맨틱 검색되지 않음 | 대화 수집과 임베딩은 별도 단계입니다. 벡터 색인 갱신 후 설정의 색인 상태를 확인합니다 |
