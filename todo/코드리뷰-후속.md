# 코드 리뷰 후속 작업

> 2026-06-10 코드 리뷰(master...HEAD) 결과와 추가 구현 아이디어.
> 완료된 항목은 체크 표시.

## 리뷰 findings (심각도순)

- [x] **1. 앱 기동 시 SetupChecker.checkAll()이 FX 스레드를 블로킹** (`LlmManagerApp.start()`)
  - 창 표시 전에 python/nvidia-smi/nvcc/node를 순차 실행 (각 최대 5초 타임아웃 + 디렉토리 스캔).
  - 조치: 기동 시에는 required 항목만 검사. Python을 required에서 제외하면서
    현재 required 항목이 없어 기동 시 프로세스 실행이 0회가 됨.

- [x] **2. `CursorAgentDialog.validateCwd()`의 `Path.of()`가 InvalidPathException 미처리**
  - 조치: try-catch로 잘못된 경로 입력으로 처리.

- [x] **3. Node/Python 설치 직후 stale PATH로 의존성 설치·실행이 계속 실패**
  - SetupChecker만 알려진 경로 폴백이 있고 PluginDependencyInstaller·InstallationService·
    ProcessManager·PluginCommandExecutor는 PATH만 봐서 앱 재시작 전까지 출구가 없었음.
  - 조치: 공용 `ToolLocator` 도입 (아이디어 1 참고).

- [x] **4. CursorAgentDialog 실행 중 취소 불가 + 창 닫아도 sidecar 잔류**
  - 조치: '중지' 버튼 추가, `executeStreaming`에 Process 노출 콜백 추가,
    창 닫기/닫기 버튼에서 프로세스 트리 강제 종료.

- [x] **5. `SetupChecker.runCommandCapture`의 reader.join(1000) 후 출력 잘림 가능**
  - 조치: join 후 `reader.isAlive()`면 잘린 출력으로 간주해 실패(null) 반환.

- [x] **6. `updateSuggestions()`가 키 입력당 2회(text+caret) 호출 + FX 스레드에서 Files.list()**
  - 조치: PauseTransition(150ms) 디바운스 적용. 디렉토리 리스팅 캐시는 에이전트가
    실행 중 파일을 생성하면 stale해질 수 있어 보류.

- [x] **7. `confirmSkipIfNeeded()`의 'Python 3' 하드코딩 문구**
  - 조치: 실패한 required 항목의 displayName을 모아 일반화된 문구로 표시.

- [x] **8. bin/*.ps1 3종에 프록시·다운로드·오프라인 폴백 블록 동일 복제 (~40줄 × 3)**
  - 조치: `bin/installer-common.ps1`로 추출, 3개 스크립트가 $PSScriptRoot 기준 dot-source.
    validate-scripts 워크플로도 공통 파일 기준으로 갱신 (트리거 브랜치 claude/** 확대).
  - 함께 수정: 배포(build.gradle distributions·prepareJpackageInput)에 install-nodejs.ps1과
    installer-common.ps1이 빠져 있던 문제 — 배포판에서 Node 설치 버튼이 동작하지 않았음.

- [x] **9. `CursorAgentDialog.listAgentFiles()`는 `listFilesByExtension(dir, ".md")`와 동일 구현**
  - 조치: listAgentFiles 삭제, listFilesByExtension 재사용.

- [ ] **10. "cursor.runAgent" 문자열 분기(MainController)·NODEJS 하드코딩(SetupItem)**
  - plugin.json에 `uiMode`/`systemRequires` 선언 → PluginManager가 라우팅하는 구조로 일반화.
    두 번째 플러그인 추가 시점에 착수 권장.
