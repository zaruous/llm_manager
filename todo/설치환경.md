# 설치 환경 체크/설치 스크립트 검토 및 개선

> 2026-06-09 검토. 대상: `LlmManagerApp.start()` → `SetupCheckDialog` → `SetupChecker`
> → `bin/install-python.ps1`, `bin/install-nvidia-driver.ps1`
>
> 처음 설치하는 사용자가 앱 실행 시 겪을 수 있는 에러·문제 사항 검토 결과.
> 완료된 항목은 체크 표시.

## 치명적 문제 (첫 사용자 차단)

- [x] **1. Python 설치 성공 후에도 재검사가 계속 '미설치' → '계속' 버튼 영구 비활성**
  - 원인: `install-python.ps1`이 PATH를 Machine/User 환경변수에 등록해도
    이미 실행 중인 Java 프로세스의 환경변수는 갱신되지 않음.
    `recheckItem()`의 `python --version`은 옛 PATH를 상속받아 계속 실패.
  - 조치: `SetupChecker.checkPython()`에 알려진 설치 경로 폴백 추가
    (`%LOCALAPPDATA%\Programs\Python\Python3xx`, `C:\Python3xx`).
    `checkCuda()`에도 `C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v*` 폴백 추가.

- [x] **2. NVIDIA 스크립트의 관리자 권한 상승이 Windows PowerShell 5.1에서 동작 안 함**
  - 원인: `Join-String`은 PowerShell 6.2+ 전용 cmdlet — 비관리자 사용자가 설치 클릭 시 즉시 에러.
    switch 파라미터 재전달 형식(`-NonInteractive True`)도 바인딩 에러.
    상승 후 부모가 즉시 exit 0 → Java가 "설치 완료"로 오인하고 로그 캡처 불가.
  - 조치: PS 5.1 호환 인수 배열 빌드, switch는 키만 전달,
    `Start-Process -Verb RunAs -Wait -PassThru`로 대기 후 자식 종료 코드 반환, UAC 거부 시 exit 1.

- [x] **3. `Download-File`의 `$wc.DownloadProgressChanged += {...}`는 PowerShell에서 유효하지 않은 문법**
  - 원인: .NET 이벤트 구독은 `Register-ObjectEvent`가 필요한 C# 문법 —
    StrictMode + ErrorActionPreference=Stop 하에서 드라이버/CUDA 다운로드가 항상 실패할 가능성.
  - 조치: `Invoke-WebRequest -UseBasicParsing`으로 교체
    (PS 5.1 진행률 바 성능 문제 회피를 위해 `$ProgressPreference = 'SilentlyContinue'`).

## 중간 수준 문제

- [x] **4. 비-Windows OS 미고려** — Linux/mac에서도 다이얼로그가 뜨고
  설치 버튼 클릭 시 `Cannot run program "powershell"` 에러.
  - 조치: `LlmManagerApp.start()`에 `PlatformUtil.isWindows()` 가드,
    `startInstall()`에도 안전 가드 추가.

- [x] **5. 매 실행마다 환경 체크 다이얼로그 강제 표시** — 모든 항목 통과 시에도 '계속' 클릭 필요.
  - 조치: 다이얼로그 표시 전 사전 검사 수행, 전 항목 통과 시 다이얼로그 생략.
    검사 결과는 다이얼로그에 시드해 중복 검사 방지.

- [x] **6. CUDA 항목의 설치 버튼이 드라이버까지 함께 설치** (수백 MB 불필요 다운로드)
  - 조치: CUDA 항목엔 `-SkipDriver`, NVIDIA 드라이버 항목엔 `-SkipCuda` 전달.

- [x] **7. PowerShell 출력 인코딩 불일치** — Java는 UTF-8로 읽지만
  Windows PowerShell 5.1 출력은 OEM 코드페이지(CP949) → 예외 메시지 한글 깨짐.
  - 조치: 두 스크립트 첫머리에 `[Console]::OutputEncoding = UTF8` 설정 (실패 시 무시).

- [x] **8. install-python.ps1 "이미 설치됨" 분기 불일치** — "5초 후 재설치 계속" 안내 후
  실제로는 exit 0. GUI 호출 컨텍스트에서 "Ctrl+C 취소" 안내 무의미 + 5초 불필요 대기.
  - 조치: 이미 설치 시 즉시 exit 0으로 단순화.

## 경미한 사항

- [x] **9. `checkPython()`이 Python 2도 통과** — 버전 출력 미파싱.
  - 조치: 출력에 "Python 3" 포함 여부 확인.

- [x] **10. `SetupChecker` Javadoc/필드 선언 순서 꼬임** (`check()` Javadoc이 `overrides` 필드에 붙음)
  - 조치: 선언 순서 정리.

- [x] **11. PATH 중복 체크 `-notlike "*$p*"` 부분 문자열 매칭 부정확**
  - 조치: `;` 분할 후 정확 비교.

- [x] **12. `TextArea`를 `ScrollPane`으로 중복 래핑** — TextArea 자체 스크롤로 충분.
  - 조치: ScrollPane 제거.

- [x] **13. '건너뜀'이 required 항목도 무경고 우회**
  - 조치: 필수 항목 미통과 상태로 건너뛸 때 확인 다이얼로그 표시.

## 프록시/오프라인 환경 대비 (2차 추가)

- [x] **14. 인증 프록시(NTLM/Kerberos) 자동 통과** — 사내 인증 프록시는 익명 요청을
  407로 거부해 다운로드가 실패.
  - 조치: 두 스크립트 시작 시 시스템 프록시에 Windows 로그온 자격증명 부착
    (`DefaultWebProxy.Credentials = DefaultNetworkCredentials`).
    `HTTP_PROXY`/`HTTPS_PROXY` 환경변수가 있으면 `Invoke-WebRequest`/`-RestMethod`에
    `-Proxy` + `-ProxyUseDefaultCredentials`로 전달.
    install-python.ps1의 WebClient도 동일 로직의 `Invoke-Download`로 교체.

- [x] **15. 오프라인 설치 파일 폴백 (완전 차단망 대비)** — `~/llm-services/installers/`에
  사전 배치된 설치 파일을 다운로드보다 우선 사용.
  - Python: 정확한 파일명 (`python-<버전>-<arch>.exe`)
  - NVIDIA 드라이버: `nvidia-driver*.exe` 패턴 (최신 파일 선택, API 조회도 생략)
  - CUDA: `cuda_<버전>*.exe` 패턴
  - 다운로드 실패 시 "URL에서 받아 → 이 경로에 배치 → 재시도" 안내를 로그에 출력.
  - 사전 배치 파일은 설치 후에도 삭제하지 않음 (TEMP 다운로드만 정리).
