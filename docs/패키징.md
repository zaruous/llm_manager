# LLM Manager — 설치 패키지 빌드 가이드

## 개요

JDK 17에 내장된 `jpackage` 도구를 이용해 **Java 설치 없이 실행 가능한** 네이티브 패키지를 생성한다.  
JRE가 앱 이미지에 번들되므로 사용자 PC에 Java가 설치되어 있지 않아도 실행된다.

---

## 요구 사항

| 항목 | 필요 버전 | 비고 |
|------|-----------|------|
| JDK | 17 이상 | `jpackage` 내장 (JDK 14+) |
| Gradle | 9.x | 프로젝트 기본 설정 |
| WiX Toolset | 3.0 이상 | **EXE/MSI 생성 시에만** 필요 |

WiX 설치 (Windows EXE/MSI 빌드 시):  
→ https://github.com/wixtoolset/wix3/releases/tag/wix3141rtm

## 빌드 명령

### app-image (zip 배포용, WiX 불필요)

```bash
./gradlew jpackageImage
```

- 출력: `build/installer/LLM Manager/`
- JRE 번들 포함 폴더 구조
- 폴더째로 zip 압축해 배포 가능 (사용자가 압축 해제 후 `.exe` 실행)
- **WiX 없어도 빌드 가능** — 첫 배포 시 권장

### EXE/DMG/DEB 인스톨러 (WiX 필요)

```bash
./gradlew jpackageExe
```

- 출력: `build/installer/LLM Manager-1.0.0.exe` (Windows 기준)
- 더블클릭으로 설치, 시작 메뉴·바탕화면 단축키 자동 생성
- Windows: WiX 3.0+ 필요
- macOS: DMG 생성
- Linux: DEB 생성

## 빌드 파이프라인

```
./gradlew jpackageImage
         │
         ├─ compileJava / processResources / jar
         │
         ├─ installDist
         │     build/install/llm-manage/
         │       lib/
         │         llm-manage-1.0.0.jar   ← 앱 JAR
         │         *.jar                  ← 의존성 JARs
         │         def/                   ← 기본 제공 서비스 정의 JSON
         │
         ├─ prepareJpackageInput
         │     build/jpackage-input/
         │       llm-manage-1.0.0.jar     (JARs + def/ 를 한 디렉토리로 복사)
         │       *.jar
         │       def/
         │
         └─ jpackageImage (jpackage 실행)
               build/installer/LLM Manager/
```

## 생성된 app-image 구조

```
build/installer/
  LLM Manager/
    LLM Manager.exe        ← 실행 파일
    app/
      LLM Manager.cfg      ← jpackage 런처 설정 (classpath, JVM 옵션)
      llm-manage-1.0.0.jar
      *.jar                 ← 모든 의존성 JAR
      def/
        bge-m3.json         ← 기본 제공 서비스 정의
        sql-gen-mcp.json
        swagger-mcp.json
    runtime/
      bin/java.exe          ← 번들된 JRE
      ...
```

`def/` 디렉토리는 앱 JAR와 같은 레벨에 위치한다.  
`BuiltinServiceLoader`가 JAR 위치 기준으로 `def/`를 자동 탐색하므로 경로를 별도 설정할 필요가 없다.

## 런처 설정 (`LLM Manager.cfg`)

jpackage가 자동 생성하는 런처 설정 파일의 핵심 항목:

```ini
[JavaOptions]
java-options=--module-path $APPDIR          # JavaFX 모듈을 app/ 디렉토리에서 로드
java-options=--add-modules javafx.controls,javafx.fxml,javafx.web,javafx.graphics,javafx.base
java-options=-Xmx512m
java-options=--add-opens java.base/java.lang=ALL-UNNAMED
```

`$APPDIR`는 jpackage 런타임 매크로로, 설치된 `app/` 디렉토리 절대 경로로 치환된다.

## 커스터마이징

### 버전 변경

`build.gradle` 상단의 `version` 값을 수정한다.

```groovy
version = '1.1.0'   // 변경
```

### 아이콘 변경

아이콘 파일을 준비한 뒤 `build.gradle`의 `--icon` 옵션 주석을 해제한다.

```
packaging/
  windows/LLM Manager.ico   (256×256 ICO)
  macos/LLM Manager.icns
  linux/LLM Manager.png     (48×48 PNG)
```

```groovy
// build.gradle — buildJpackageArgs() 내
'--icon', "packaging/windows/LLM Manager.ico",  // 주석 해제
```

### JVM 메모리 조정

```groovy
'--java-options', '-Xmx512m',   // 기본값, 필요에 따라 변경
```

### MSI 생성 (Windows)

`jpackageExe` 태스크의 type을 `'msi'`로 변경하면 MSI 패키지를 생성한다.

```groovy
tasks.register('jpackageExe', Exec) {
    doFirst {
        commandLine buildJpackageArgs('msi', project)   // 'exe' → 'msi'
    }
}
```

## 주의 사항

### lib/*.jar는 Git에 포함되지 않음

번들 MCP 서버 JAR(`lib/sql-gen-mcp-*.jar`, `lib/swagger-mcp-server.jar`)는  
GitHub 100MB 제한 초과로 `.gitignore`에 등록되어 있다.  
패키지 빌드 전 해당 JAR 파일이 `lib/` 디렉토리에 있는지 확인한다.

```
lib/
  sql-gen-mcp-1.0.0-SNAPSHOT.jar    ← Git 미포함, 수동 배치 필요
  swagger-mcp-server.jar            ← Git 미포함, 수동 배치 필요
  def/
    *.json                          ← Git 포함
```

### 크로스 플랫폼 빌드 불가

jpackage는 **현재 OS 전용** 패키지만 생성한다.  
Windows EXE는 Windows에서, macOS DMG는 macOS에서 빌드해야 한다.

### build/installer/ 는 .gitignore에 추가 권장

```gitignore
# .gitignore 에 추가
build/
```

`build/` 디렉토리는 이미 `.gitignore`에 포함되어 있으므로 별도 추가 불필요.
