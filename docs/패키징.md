# LLM Manager — 패키지 빌드 가이드

## 개요

JDK 17에 내장된 `jpackage` 도구를 이용해 **Java 설치 없이 실행 가능한** 네이티브 패키지를 생성한다.  
JRE가 앱 이미지에 번들되므로 사용자 PC에 Java가 설치되어 있지 않아도 실행된다.

---

## 빠른 시작

```powershell
# 실행 파일 빌드 (기본 — WiX 불필요)
powershell -ExecutionPolicy Bypass -File bin\buildExe.ps1
# → deploy\LLMManager-1.0.0.zip  (압축 해제 → LLMManager.exe 바로 실행)

# 설치 파일 빌드 (WiX 자동 다운로드)
powershell -ExecutionPolicy Bypass -File bin\buildExe.ps1 -Installer
# → deploy\LLMManager-1.0.0.exe  (더블클릭 설치)
```

---

## 배포 유형 비교

| 구분 | 실행 파일 (기본) | 설치 파일 (`-Installer`) |
|------|-----------------|-------------------------|
| 출력 | `deploy\LLMManager-x.x.x.zip` | `deploy\LLMManager-x.x.x.exe` |
| 사용법 | 압축 해제 → `LLMManager.exe` 실행 | 더블클릭 → 설치 마법사 |
| 설치 과정 | 없음 | 시작 메뉴·바탕화면 단축키 자동 생성 |
| WiX 필요 | **불필요** | 필요 (자동 다운로드) |
| Gradle 태스크 | `jpackageImage` | `jpackageExe` |

---

## 요구 사항

| 항목 | 필요 버전 | 비고 |
|------|-----------|------|
| JDK | 17 이상 | `jpackage` 내장 (JDK 14+) |
| Gradle | 9.x | 프로젝트 기본 설정 |
| WiX Toolset | 3.0 이상 | **설치 파일(`-Installer`) 빌드 시에만** 필요 |

WiX는 `buildExe.ps1 -Installer` 실행 시 `deploy\WiX\`에 자동 다운로드된다.  
수동 설치: https://github.com/wixtoolset/wix3/releases/tag/wix3141rtm

## buildExe.ps1 파이프라인

### 실행 파일 모드 (기본)

```
bin\buildExe.ps1
  │
  ├─ [1/2] ./gradlew jpackageImage
  │         ├─ installDist
  │         │     build/install/llm-manage/lib/  (JARs + def/)
  │         ├─ prepareJpackageInput
  │         │     build/jpackage-input/          (JARs + def/ 평탄화)
  │         └─ jpackage --type app-image
  │               build/installer/LLMManager/    (LLMManager.exe + JRE 번들)
  │
  └─ [2/2] app-image → ZIP 압축
            deploy\LLMManager-1.0.0.zip
```

### 설치 파일 모드 (`-Installer`)

```
bin\buildExe.ps1 -Installer
  │
  ├─ [1/3] WiX 확인 / 다운로드
  │         deploy\WiX\candle.exe, light.exe ...
  │
  ├─ [2/3] ./gradlew jpackageExe
  │         └─ jpackage --type exe
  │               build/installer/LLMManager-1.0.0.exe
  │
  └─ [3/3] EXE → deploy\ 복사
            deploy\LLMManager-1.0.0.exe
```

## Gradle 태스크 직접 실행

`buildExe.ps1` 없이 Gradle 태스크를 직접 실행할 수도 있다.

```bash
# 실행 파일 (app-image 폴더)
./gradlew jpackageImage
# → build/installer/LLMManager/

# 설치 파일 (WiX 가 PATH 에 있어야 함)
./gradlew jpackageExe
# → build/installer/LLMManager-1.0.0.exe
```

## 생성된 app-image 구조

```
build/installer/
  LLMManager/
    LLMManager.exe          ← 실행 파일 (설치 불필요)
    app/
      LLMManager.cfg        ← jpackage 런처 설정 (classpath, JVM 옵션)
      llm-manage-1.0.0.jar
      *.jar                  ← 모든 의존성 JAR
      def/
        bge-m3.json          ← 기본 제공 서비스 정의
        sql-gen-mcp.json
        swagger-mcp.json
    runtime/
      bin/java.exe           ← 번들된 JRE (Java 설치 불필요)
      ...
```

`def/` 디렉토리는 앱 JAR와 같은 레벨에 위치한다.  
`BuiltinServiceLoader`가 JAR 위치 기준으로 `def/`를 자동 탐색하므로 경로 설정 불필요.

---

## 런처 설정 (`LLMManager.cfg`)

jpackage가 자동 생성하는 런처 설정 파일의 핵심 항목:

```ini
[JavaOptions]
java-options=--module-path $APPDIR          # JavaFX 모듈을 app/ 에서 로드
java-options=--add-modules javafx.controls,javafx.fxml,javafx.web,javafx.graphics,javafx.base
java-options=-Xmx512m
java-options=--add-opens java.base/java.lang=ALL-UNNAMED
```

`$APPDIR`는 jpackage 런타임 매크로로, 실행 시 `app/` 디렉토리 절대 경로로 치환된다.

## 커스터마이징

### 버전 변경

`build.gradle` 상단의 `version` 값을 수정한다.

```groovy
version = '1.1.0'
```

### 아이콘 변경

아이콘 파일을 준비한 뒤 `build.gradle`의 `--icon` 옵션 주석을 해제한다.

```
packaging/
  windows/LLMManager.ico    (256×256 ICO)
  macos/LLMManager.icns
  linux/LLMManager.png      (48×48 PNG)
```

```groovy
// build.gradle — buildJpackageArgs() 내
'--icon', "packaging/windows/LLMManager.ico",  // 주석 해제
```

### JVM 메모리 조정

```groovy
'--java-options', '-Xmx512m',   // 기본값, 필요에 따라 변경
```

## 주의 사항

### --description 에 한글 사용 금지

`build.gradle`의 `--description` 값에 한글을 쓰면 candle.exe(WiX 컴파일러)가  
깨진 문자로 받아 light.exe가 exit 311 오류를 낸다. **반드시 영문**으로 작성한다.

```groovy
'--description', 'LLM Service Manager',  // 영문 필수
```

### lib/*.jar 는 Git 에 포함되지 않음

번들 MCP 서버 JAR(`lib/sql-gen-mcp-*.jar`, `lib/swagger-mcp-server.jar`)는  
GitHub 100MB 제한 초과로 `.gitignore`에 등록되어 있다.  
패키지 빌드 전 해당 JAR 파일이 `lib/` 디렉토리에 있는지 확인한다.

```
lib/
  sql-gen-mcp-1.0.0-SNAPSHOT.jar    ← Git 미포함, 수동 배치 필요
  swagger-mcp-server.jar            ← Git 미포함, 수동 배치 필요
  def/
    *.json                          ← Git 포함
```

### 크로스 플랫폼 빌드 불가

jpackage는 **현재 OS 전용** 패키지만 생성한다.  
Windows EXE는 Windows에서, macOS DMG는 macOS에서 빌드해야 한다.
