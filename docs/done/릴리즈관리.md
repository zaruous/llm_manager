# 릴리즈 관리 계획

## 목표

GitHub Releases를 통해 앱 업데이트를 배포하고,
사용자가 설정 메뉴에서 직접 버전을 확인·설치할 수 있도록 한다.
업데이트는 수동 트리거 전용 (자동 확인 없음).

---

## 배포 산출물

포터블 앱 이미지(`jpackageImage`) 기준으로 배포한다.
인스톨러(.msi) 없이 디렉토리 전체를 zip으로 묶어 GitHub Release에 첨부.

```
git tag v1.x.x
  → GitHub Release 생성
  → LLMManager-1.x.x.zip 첨부  (build/installer/LLMManager/ 전체를 zip)
```

---

## 업데이트 흐름

```
[업데이트 확인 클릭]
  → GitHub API /releases/latest 호출
  → 버전 비교
      ├─ 최신: "현재 최신 버전입니다" Alert
      └─ 신규 버전 있음: UpdateInstallDialog 표시

[UpdateInstallDialog — 설치 클릭]
  → zip 다운로드 → %TEMP%\llm-manager-update\LLMManager-x.x.x.zip
  → update.bat 생성 → %TEMP%\llm-manager-update\update.bat
  → update.bat 실행 (새 프로세스, 분리 실행)
  → Platform.exit() 로 현재 앱 종료

[update.bat 동작 — 앱 종료 후]
  1. 3초 대기 (앱 프로세스 완전 종료 보장)
  2. PowerShell Expand-Archive 로 zip 압축 해제
  3. xcopy /E /Y 로 현재 설치 디렉토리에 덮어쓰기
  4. 새 버전 LLMManager.exe 재실행
  5. 임시 파일 정리
```

---

## update.bat 구조

```bat
@echo off
chcp 65001 > nul
timeout /t 3 /nobreak > nul

set INSTALL_DIR=%~1
set ZIP_PATH=%~2
set EXTRACT_DIR=%TEMP%\llm-manager-update\extracted

powershell -Command "Expand-Archive -Path '%ZIP_PATH%' -DestinationPath '%EXTRACT_DIR%' -Force"
xcopy /E /Y "%EXTRACT_DIR%\LLMManager\*" "%INSTALL_DIR%\"

start "" "%INSTALL_DIR%\LLMManager.exe"

rmdir /S /Q "%EXTRACT_DIR%"
```

- `%~1` = 현재 설치 디렉토리 (Java에서 `ProcessHandle.current()` 경로로 산출)
- `%~2` = 다운로드된 zip 경로
- PowerShell `Expand-Archive` 사용 (Windows 10+ 기본 내장)

## 기술 메모

| 항목 | 결정 |
|------|------|
| 배포 포맷 | zip (포터블 앱 이미지, 인스톨러 없음) |
| API 인증 | public repo → 토큰 불필요 (rate limit 60 req/h) |
| 버전 소스 | `build.gradle` `version` → `app.properties` 빌드 타임 주입 |
| 버전 비교 | 직접 구현 (x.y.z 정수 비교, 외부 라이브러리 미사용) |
| 다운로드 | Java `HttpClient` (JDK 11+) |
| 파일 교체 | `update.bat` — PowerShell `Expand-Archive` + `xcopy` |
| 설치 경로 산출 | `ProcessHandle.current().info().command()` |
| private repo 전환 시 | `app.properties`에 `update.token` 추가 |
