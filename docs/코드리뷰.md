# 코드리뷰

리뷰 대상: `D:\git\java\llm_manager` 현재 워크트리  
검증: `.\gradlew.bat build` 성공, 테스트 실행/통과

## 주요 발견 사항

### [v] High: 내장 API 서버가 인증 없이 서비스 제어 API를 노출함

- 위치: `src/main/java/org/kyj/llmmanager/service/EmbeddedApiServer.java:59`, `src/main/java/org/kyj/llmmanager/service/EmbeddedApiServer.java:62`, `src/main/java/org/kyj/llmmanager/service/EmbeddedApiServer.java:99`
- 내용: API 서버가 켜지면 `/api/services/{id}/start`, `/stop`, `/restart`가 인증/토큰/로컬 요청 검증 없이 등록된다. 동시에 CORS도 `anyHost()`로 열려 있다.
- 영향: 사용자가 API 서버를 활성화한 상태에서 포트가 외부에 노출되면, 브라우저나 외부 프로세스가 등록 서비스의 시작/중지를 원격으로 호출할 수 있다. 서비스 정의에는 실행 명령과 환경변수도 포함되므로 운영상 영향이 크다.
- 권고: 기본 바인딩을 loopback으로 제한하고, 제어 API에는 설정 가능한 토큰 또는 로컬 요청 검증을 추가한다. CORS는 Swagger UI 등 필요한 origin으로 제한한다.

### [v] High: SQLite 외 DB provider를 설정하면 런타임에서 바로 실패할 가능성이 큼

- 위치: `src/main/java/org/kyj/llmmanager/service/LlmSkillLibraryRepository.java:372`, `src/main/java/org/kyj/llmmanager/service/LlmSkillLibraryRepository.java:397`, `build.gradle:32`
- 내용: 설정 모델은 `postgresql`, `oracle`, `mssql` provider를 허용하고 각 드라이버 클래스를 반환하지만, 빌드 의존성에는 `sqlite-jdbc`만 포함되어 있다.
- 영향: 사용자가 SQLite 외 provider를 선택하면 Hikari 초기화 또는 첫 연결 시점에 `ClassNotFoundException`/드라이버 없음 오류가 발생한다. UI에서는 provider가 지원되는 것처럼 보이는데 실제 배포물은 동작하지 않는다.
- 권고: 지원 provider만 UI/설정에서 노출하거나, PostgreSQL/MSSQL 드라이버를 런타임 의존성에 추가한다. Oracle은 라이선스/배포 정책을 별도 처리해야 한다.

### [v] Medium: API 오류/성공 응답이 JSON 객체가 아니라 JSON 문자열로 반환될 수 있음

- 위치: `src/main/java/org/kyj/llmmanager/service/EmbeddedApiServer.java:133`, `src/main/java/org/kyj/llmmanager/service/EmbeddedApiServer.java:144`, `src/main/java/org/kyj/llmmanager/service/EmbeddedApiServer.java:148`
- 내용: `ctx.json("{\"error\":\"...\"}")`처럼 문자열을 `json()`에 넘기고 있다.
- 영향: 클라이언트가 기대하는 `{ "error": "..." }` 객체가 아니라 JSON 문자열로 직렬화될 수 있어 Swagger/외부 클라이언트 연동에서 파싱 형태가 흔들린다.
- 권고: `Map.of("error", "...")` 또는 `ObjectNode`를 넘기는 방식으로 통일한다.

### [v] Medium: 서비스 선택/대시보드 재빌드 때 리스너가 누적됨

- 위치: `src/main/java/org/kyj/llmmanager/ui/controller/MainController.java:301`, `src/main/java/org/kyj/llmmanager/ui/controller/MainController.java:1039`, `src/main/java/org/kyj/llmmanager/ui/controller/MainController.java:1136`, `src/main/java/org/kyj/llmmanager/ui/controller/MainController.java:1139`
- 내용: 서비스 선택 시 `statusProperty` 리스너를 매번 추가하지만 제거하지 않는다. 대시보드 카드를 다시 만들 때도 `memoryBytesProperty`, `statusProperty` 리스너가 매번 새로 붙는다.
- 영향: 화면 전환/새로고침/상세 수정이 반복될수록 같은 상태 변경에 여러 콜백이 실행되고, 이전 카드의 UI 객체가 리스너에 붙어 GC되지 않을 수 있다. 장시간 실행 앱에서는 메모리 증가와 UI 갱신 중복으로 이어진다.
- 권고: 선택 상태 리스너는 필드로 보관해 이전 선택에서 제거한다. 대시보드 카드는 바인딩 해제 가능한 구조로 만들거나 카드 재빌드 대신 기존 노드를 갱신한다.

### [v] Medium: 명령어 파싱이 실제 셸 규칙과 달라 경로/인수 처리 오류가 날 수 있음

- 위치: `src/main/java/org/kyj/llmmanager/util/CommandBuilder.java:58`, `src/main/java/org/kyj/llmmanager/service/InstallationService.java:150`, `src/main/java/org/kyj/llmmanager/ui/controller/MainController.java:766`
- 내용: 공백 기준 수동 파싱이 여러 곳에 흩어져 있다. 따옴표는 일부 처리하지만 escape, 백슬래시, 빈 인수, 중첩 인용, Windows 경로 등은 온전히 처리하지 못한다.
- 영향: 설치 경로나 JAR 경로에 공백/특수문자가 들어가면 실행 파일 판별, 번들 JAR 탐색, 실제 프로세스 실행이 서로 다르게 동작할 수 있다.
- 권고: 명령을 문자열 하나로 저장하기보다 실행 파일과 인수 리스트를 구조화한다. 불가피하면 Apache Commons Exec 등 검증된 파서를 사용하고 파싱 유닛 테스트를 추가한다.

### [v] Low: 테스트 소스가 없어 핵심 동작이 빌드에서 검증되지 않음

- 위치: 빌드 결과 `compileTestJava NO-SOURCE`, `test NO-SOURCE`
- 내용: 프로세스 제어, API 응답, 설정 저장, 명령어 파싱, 스킬 설치 경로 같은 회귀 위험 영역에 테스트가 없다.
- 영향: 현재처럼 UI/서비스 계층에 상태 전이와 파일/프로세스 작업이 섞여 있으면 수동 확인 없이는 작은 변경도 회귀를 만들기 쉽다.
- 권고: 우선 `CommandBuilder`, `LlmSkillLibraryRepository`의 provider별 동작, `EmbeddedApiServer` 응답 형태를 작은 단위 테스트로 고정한다.

## 기타 관찰

- [v] `AppContext.shutdown()`은 `processManager.stopAll()`을 비동기로 호출한 뒤 곧바로 `logService.shutdown()`을 호출한다. 종료 로그 유실이나 stop 완료 전 앱 종료 가능성이 있다.
- [v] `LlmSkillLibraryRepository.TABLE_NAME`은 선언만 되고 사용되지 않는다.
- [v] 워크트리에 기존 수정 파일과 산출물(`deploy/`, `lib/cursor/`)이 존재한다. 이번 리뷰 문서 외에는 변경하지 않았다.

## 반영 사항

### 2026-06-07: 내장 API 서버 서비스 제어 API 보호

- API 서버 기본 바인딩 호스트를 `127.0.0.1`로 추가해 기본 상태에서 외부 네트워크 접근을 차단했다.
- `Javalin` CORS `anyHost()` 허용을 제거했다.
- `/api/services/{id}/start`, `/api/services/{id}/stop`, `/api/services/{id}/restart`에 제어 토큰 검사를 추가했다.
- 토큰은 `X-LLM-Manager-Token` 헤더 또는 `Authorization: Bearer <token>` 형식으로 전달할 수 있다.
- `application.yml`에 다음 설정을 추가했다.
  - `api.server.host`
  - `api.server.token`
  - `api.server.allow-unauthenticated-control`
- `api.server.allow-unauthenticated-control=true`이면 서비스 제어 API를 토큰 없이 허용하도록 했다. 기본값은 `false`다.
- 환경 설정 화면에 API 호스트, 제어 토큰, 토큰 생성 버튼, 인증 없이 제어 허용 체크박스를 추가했다.
- Swagger/OpenAPI에 `X-LLM-Manager-Token` API key 보안 스키마와 401 응답을 반영했다.
- 검증: `.\gradlew.bat build` 성공. 테스트 소스는 없어 `test NO-SOURCE`.

### 2026-06-07: SQLite 외 DB provider JDBC 드라이버 추가

- `LlmSkillLibraryRepository`가 허용하는 `postgresql`, `mssql`, `oracle` provider에 맞춰 런타임 JDBC 드라이버 의존성을 추가했다.
- 추가 의존성:
  - `org.postgresql:postgresql:42.7.10`
  - `com.microsoft.sqlserver:mssql-jdbc:13.4.0.jre11`
  - `com.oracle.database.jdbc:ojdbc11:23.26.2.0.0`
- SQLite는 기존 `org.xerial:sqlite-jdbc:3.45.2.0`을 유지했다.
- 검증: `.\gradlew.bat build` 성공. 테스트 소스는 없어 `test NO-SOURCE`.

### 2026-06-07: API 오류/성공 응답 JSON 객체화

- `ctx.json("{\"error\":\"...\"}")`, `ctx.json("{\"result\":\"...\"}")`처럼 JSON 문자열을 넘기던 응답을 제거했다.
- `errorResponse(ctx, status, message)`와 `resultResponse(ctx, message)` 헬퍼를 추가해 `Map` 기반 JSON 객체로 응답하도록 통일했다.
- 적용 대상:
  - 서비스 없음 404 응답
  - 이미 실행 중/실행 중 아님 409 응답
  - start/stop/restart 요청 성공 응답
  - 직렬화 오류 500 응답
- 검증: 문자열 JSON 응답 패턴 검색 결과 없음. `.\gradlew.bat build` 성공. 테스트 소스는 없어 `test NO-SOURCE`.

### 2026-06-07: 서비스 선택/대시보드 리스너 누적 방지

- 선택 서비스 상세 화면의 로그/상태 리스너를 `detachSelectedServiceListeners()`로 공통 해제하도록 정리했다.
- 서비스 선택 변경, 선택 해제, 서비스 삭제 시 이전 선택 서비스에 붙은 리스너를 제거한다.
- 대시보드 카드 재빌드 전에 이전 카드가 등록한 `memoryBytesProperty`, `statusProperty` 리스너를 `clearDashboardCardListeners()`로 해제하도록 했다.
- 메모리 상세 팝업의 `memoryBytesProperty` 리스너도 팝업이 닫힐 때 제거하도록 했다.
- 검증: `.\gradlew.bat build` 성공. 테스트 소스는 없어 `test NO-SOURCE`.
